# NexusAI Project Guide

## Product Context

Build NexusAI as an AI model discovery and agent-building platform based on `requirements.md`.

Core scope:

- Guided onboarding and discovery
- Chat Hub for personalized recommendations
- AI model marketplace with search, filtering, and comparison
- Model detail pages with pricing, prompt guides, and reviews
- Agent Builder with templates and guided setup
- Dashboard with usage metrics
- Research feed and localization support

## Tech Stack

- Frontend: Next.js, Tailwind CSS, TypeScript
- Backend: NestJS, TypeScript
- Database: MongoDB

## Coding Rules

- Follow DRY at all times
- Use functional components only on the frontend
- Use TypeScript types for everything
- Prefer small, composable modules over large files
- Keep business logic out of UI components
- Validate all API inputs on the backend
- Use consistent naming across frontend, backend, and database models
- Favor clear interfaces, DTOs, and typed response contracts

## Architecture Guidance

### Frontend

- Use Next.js App Router unless a strong reason requires otherwise
- Use Tailwind utility classes with reusable UI primitives
- Keep pages thin and move logic into hooks, services, and typed helpers
- Use server/client boundaries intentionally
- Represent all API payloads with explicit shared types where practical

### Backend

- Organize NestJS by domain modules
- Use controllers, services, DTOs, and schema/model layers consistently
- Keep controllers thin and place business logic in services
- Use MongoDB collections aligned to product domains such as users, models, agents, reviews, prompts, and research items

### Data and Integration

- Design APIs to support marketplace listing, model detail, comparison, onboarding answers, prompt generation metadata, agent templates, and dashboard metrics
- Keep integration contracts versionable and typed
- Plan for eventual auth, but do not overbuild features that are marked partial or not evident in `requirements.md`

## Agent Roles

### FE Agent

Responsibilities:

- Own Next.js and Tailwind implementation
- Build pages, layouts, components, forms, and interaction flows
- Implement responsive UX for onboarding, marketplace, chat hub, model detail, dashboard, and agent builder
- Maintain design consistency and accessibility
- Consume backend APIs through typed clients

Deliverables:

- Functional UI
- Typed components and hooks
- Loading, empty, and error states

### BE Agent

Responsibilities:

- Own NestJS services, controllers, DTOs, schemas, and database access
- Design and implement APIs for models, comparisons, prompts, agents, reviews, research feed, and dashboard metrics
- Enforce validation, typing, and domain boundaries
- Prepare MongoDB data models and persistence flows

Deliverables:

- Stable typed APIs
- Domain services
- Validation and persistence layers

### Integration Agent

Responsibilities:

- Coordinate frontend and backend contracts
- Align request/response payloads, error handling, and environment configuration
- Ensure pages are wired to live APIs correctly
- Verify end-to-end flows across onboarding, discovery, model browsing, comparison, and agent creation
- Catch schema mismatches and integration regressions early

Deliverables:

- Shared contract alignment
- End-to-end integration checks
- Cross-layer bug resolution

### Sprint Planner Agent

Responsibilities:

- Translate `requirements.md` into epics, stories, and implementation tasks
- Manage Jira ticket structure and sprint-ready backlog
- Prioritize work using MoSCoW from the requirements
- Break large features into FE, BE, and integration subtasks
- Track dependencies, blockers, and acceptance criteria

Deliverables:

- Jira-ready backlog
- Sprint scope proposals
- Clear ticket descriptions with acceptance criteria

## Working Style

- Keep responses concise
- Only provide code blocks unless explanation is requested
- Do not add unnecessary commentary
- Implement only what is in current scope
- Treat unconfirmed features from the prototype as optional until explicitly approved

## Priority Implementation Order

1. Guided onboarding and discovery
2. Marketplace search, filters, and model listing
3. Model detail and comparison
4. Chat Hub recommendation flow
5. Agent Builder
6. Dashboard metrics
7. Research feed
8. Localization enhancements

## Definition of Done

- Feature matches `requirements.md`
- Types are complete across the implementation
- UI is responsive and accessible
- API contracts are validated
- Integration between frontend and backend is verified
- Code remains DRY and modular
